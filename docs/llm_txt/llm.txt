# Kestra - Event-Driven Declarative Orchestration Platform

## Project Overview

Kestra is an open-source, event-driven orchestration platform that makes both scheduled and event-driven workflows easy. It follows Infrastructure as Code best practices for data, process, and microservice orchestration, allowing users to build reliable workflows directly from the UI in just a few lines of YAML.

**Website**: https://kestra.io
**Repository**: https://github.com/kestra-io/kestra
**License**: Apache License 2.0

## Key Features

- **Everything as Code and from the UI**: Keep workflows as code with Git Version Control integration, even when building them from the UI
- **Event-Driven & Scheduled Workflows**: Automate both scheduled and real-time event-driven workflows via simple trigger definitions
- **Declarative YAML Interface**: Define workflows using simple configuration in the built-in code editor
- **Rich Plugin Ecosystem**: Hundreds of built-in plugins to extract data from any database, cloud storage, or API, and run scripts in any language
- **Intuitive UI & Code Editor**: Build and visualize workflows directly from the UI with syntax highlighting, auto-completion and real-time syntax validation
- **Scalable**: Designed to handle millions of workflows, with high availability and fault tolerance
- **Version Control Friendly**: Write workflows from the built-in code Editor and push them to your preferred Git branch directly from Kestra

## Core Concepts

### Flows
Flows are the main building blocks in Kestra, defined in YAML format. Each flow contains:
- **id**: Unique identifier
- **namespace**: Organizational grouping
- **tasks**: List of tasks to execute
- **triggers**: Optional triggers for automatic execution
- **inputs**: Parameters that can be passed to the flow
- **outputs**: Values returned from the flow

Example flow structure:
```yaml
id: example_flow
namespace: dev
tasks:
  - id: task_name
    type: io.kestra.plugin.core.log.Log
    message: "Hello, World!"
```

### Tasks
Tasks are individual units of work within a flow. Core task types include:
- **ExecutableTask**: Tasks that can be executed
- **FlowableTask**: Tasks that can contain sub-tasks
- **RunnableTask**: Tasks that run external processes
- **GenericTask**: Generic task implementation

### Triggers
Triggers define when flows should be executed:
- **Schedule**: Time-based triggers (cron expressions)
- **Webhook**: HTTP-triggered flows
- **Event**: External event-triggered flows
- **Flow**: Triggered by other flow completions

### Executions
Executions represent instances of flow runs, tracking:
- State progression (CREATED, RUNNING, SUCCESS, FAILED)
- Task outputs and logs
- Duration and metrics
- Error handling and retries

## Plugin System

Kestra's plugin system is central to its extensibility:
- **Plugin Registry**: Central registry for plugin discovery
- **Plugin Manager**: Handles plugin loading and lifecycle
- **Plugin Artifacts**: Maven-based plugin distribution
- **Plugin Documentation**: Auto-generated documentation

### Plugin Structure
Plugins are Java libraries that extend Kestra's capabilities:
- Located in `io.kestra.plugin.*` packages
- Implement specific interfaces (Task, Trigger, etc.)
- Use annotations for configuration (@Schema, @PluginProperty)
- Support for any JVM language

## Development Setup

### Building the Project
```bash
# Clone the repository
git clone https://github.com/kestra-io/kestra.git
cd kestra

# Build with Gradle
./gradlew build

# Run tests
./gradlew test

# Start development server
./gradlew server --args="server local"
```

### Frontend Development
```bash
cd ui
npm install
npm run dev  # Start development server
npm run build  # Build for production
```

### Docker Development
```bash
# Start with Docker Compose
docker-compose up -d

# Or use the single container
docker run -p 8080:8080 kestra/kestra:latest server local
```
