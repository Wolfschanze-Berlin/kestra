# Kestra Complete Plugin Ecosystem

Kestra has a vast ecosystem of 80+ plugin repositories containing hundreds of task types for every integration scenario.

## Core Built-in Plugins (107+ tasks)

### Flow Control Plugins
- **Sequential** - Execute tasks in sequence
- **Parallel** - Execute tasks concurrently
- **ForEach/EachSequential/EachParallel** - Iteration patterns
- **If** - Conditional execution
- **Switch** - Multi-branch conditional logic
- **Subflow** - Call other workflows
- **Dag** - Directed Acyclic Graph dependencies
- **LoopUntil** - Loop with exit condition
- **AllowFailure** - Optional task execution
- **Template** - Reusable task templates
- **WorkingDirectory** - File workspace management

### Utility and System Plugins
- **Debug.Return** - Output debugging information
- **Log** - Structured logging
- **Sleep** - Delays and waiting
- **Pause** - Manual approval gates
- **State.Set/Get/Delete** - State management
- **Execution.Count/Fail/Exit/Labels/Resume** - Execution control
- **Output.Values** - Output management
- **Metric.Publish** - Custom metrics

### Storage and File Plugins
- **Storage.Delete/Download/Upload** - File operations
- **Namespace.Files** - Namespace file management

### HTTP and API Plugins
- **HTTP.Request** - HTTP API calls
- **HTTP.Download** - File downloads

### Key-Value Store Plugins
- **KV.Set/Get/Delete/GetKeys** - Key-value operations

### Trigger Plugins
- **Schedule** - Cron-based scheduling
- **ScheduleOnDates** - Specific date scheduling
- **Flow** - Workflow triggers
- **Webhook** - HTTP webhook triggers
- **Toggle** - Enable/disable triggers

### Runner and Script Plugins
- **Runner.Process** - External process execution
- **Script execution** - Python, Node.js, Shell, R, etc.

## Integration Patterns

### ETL Pipeline Example
```yaml
id: comprehensive-etl
namespace: data.pipeline

tasks:
  # Extract from multiple sources
  - id: extract-postgres
    type: io.kestra.plugin.jdbc.postgresql.Query
    sql: SELECT * FROM sales WHERE date >= '{{ inputs.start_date }}'
    
  - id: extract-mongodb
    type: io.kestra.plugin.mongodb.Find
    database: analytics
    collection: events
    
  # Transform data
  - id: transform-data
    type: io.kestra.plugin.scripts.python.Script
    script: |
      import pandas as pd
      
      # Combine and transform data
      sales_df = pd.read_json('{{ outputs.extract-postgres.uri }}')
      events_df = pd.read_json('{{ outputs.extract-mongodb.uri }}')
      
      # Data processing logic here
      result = sales_df.merge(events_df, on='customer_id')
      result.to_json('output.json')
      
  # Load to multiple destinations
  - id: load-to-bigquery
    type: io.kestra.plugin.gcp.bigquery.Load
    from: "{{ outputs.transform-data.outputFiles['output.json'] }}"
    destinationTable: project.dataset.processed_data
    
  - id: load-to-s3
    type: io.kestra.plugin.aws.s3.Upload
    from: "{{ outputs.transform-data.outputFiles['output.json'] }}"
    bucket: data-lake
    key: processed/{{ execution.startDate | date('yyyy/MM/dd') }}/data.json
    
  # Notify completion
  - id: notify-completion
    type: io.kestra.plugin.notifications.slack.SlackExecution
    channel: "#data-team"
    customMessage: "ETL pipeline completed: {{ outputs.transform-data.vars.record_count }} records processed"
```

### Multi-Cloud Data Pipeline
```yaml
id: multi-cloud-pipeline
namespace: cloud.integration

tasks:
  # Source: AWS S3
  - id: download-from-s3
    type: io.kestra.plugin.aws.s3.Download
    bucket: source-data
    key: daily/data.csv
    
  # Process with Databricks
  - id: process-with-databricks
    type: io.kestra.plugin.databricks.jobs.CreateJobRun
    notebookTask:
      notebookPath: /notebooks/data_processor
      baseParameters:
        input_file: "{{ outputs.download-from-s3.uri }}"
        
  # Store in Google Cloud Storage
  - id: upload-to-gcs
    type: io.kestra.plugin.gcp.gcs.Upload
    from: "{{ outputs.process-with-databricks.outputFiles['result.parquet'] }}"
    bucket: processed-data
    blob: processed/{{ execution.startDate | date('yyyy-MM-dd') }}/result.parquet
    
  # Update Azure SQL Database
  - id: update-azure-sql
    type: io.kestra.plugin.jdbc.mssql.Query
    url: "{{ secret('AZURE_SQL_CONNECTION') }}"
    sql: |
      MERGE target_table AS target
      USING (SELECT * FROM OPENROWSET(...)) AS source
      ON target.id = source.id
      WHEN MATCHED THEN UPDATE SET ...
      WHEN NOT MATCHED THEN INSERT ...
```
